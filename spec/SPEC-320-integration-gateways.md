# SPEC-320: Интеграционные шлюзы (Integration Gateways)

**Статус:** Approved v1.0 (2026-02-02)  
**Зависимости:** SPEC-010, SPEC-140, SPEC-300  
**Назначение:** определить, как внешние системы интегрируются с overlay-сетью через gateway, не нарушая trust boundaries.

---

## 1) Принцип v1

Gateway — отдельный процесс, который:

* принимает внешние запросы (HTTP);
* преобразует их в overlay Envelope;
* возвращает ответы обратно во внешний протокол.

Gateway **НЕ ДОЛЖЕН** иметь прямого доступа к приватным ключам Identity peer, кроме как через ограниченный IPC.

В v1 gateway **МОЖЕТ** отсутствовать (отложенная реализация). Если gateway запущен, он **ДОЛЖЕН** соответствовать требованиям этого SPEC.

---

## 2) Локальный IPC (фиксировано)

Peer runtime **ДОЛЖЕН** предоставлять локальный gRPC интерфейс `PeerControl.v1`:

* на Unix: через Unix domain socket `run/peer.sock`
* на Windows: через TCP `127.0.0.1:17777`

Этот интерфейс **ДОЛЖЕН** требовать аутентификацию “локальным токеном”:

* токен хранится в `run/peer.token` (случайные 32 bytes, base64) и читается только владельцем процесса.

Требования к безопасности файла токена (фиксировано):

* токен-файл **ДОЛЖЕН** быть доступен на чтение только владельцу процесса (owner-only).
  * на POSIX: режим **ДОЛЖЕН** быть `0600`;
  * на Windows: ACL **ДОЛЖЕН** разрешать чтение только текущему пользователю/сервисному аккаунту.
* если обеспечить owner-only права не удалось, peer **ДОЛЖЕН** отказать в запуске gateway/IPC с ошибкой `ERR_GATEWAY_UNAUTHORIZED`.

Rotation (фиксировано):

* токен **ДОЛЖЕН** ротироваться при каждом старте peer runtime (новый случайный токен, старый становится невалидным).
* при старте gateway **ДОЛЖЕН** логировать `warn`, что внешний bind запрещён (loopback-only) и что token является “секретом”.

---

## 3) Внешний HTTP API (фиксировано)

Gateway **ДОЛЖЕН** предоставлять HTTP/JSON API на `127.0.0.1:8080` (по умолчанию).

Аутентификация внешнего API в v1:

* обязательный заголовок `Authorization: Bearer <token>`
* token берётся из `run/peer.token`

Экспозиция наружу (не на loopback) **НЕ ДОПУСКАЕТСЯ** в v1.

Примечание (UX v1):

* gateway — отдельный процесс и по умолчанию **не запускается** автоматически; запуск — явное действие оператора.

---

## 4) Ошибки (минимум)

* `ERR_GATEWAY_UNAUTHORIZED`
* `ERR_GATEWAY_FORBIDDEN_BIND`
* `ERR_GATEWAY_BAD_REQUEST`

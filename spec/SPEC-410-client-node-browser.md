# SPEC-410: Клиент Node Browser (минимум v1)

**Статус:** Approved v1.0 (2026-02-02)  
**Зависимости:** SPEC-200, SPEC-210, SPEC-220, SPEC-420  
**Назначение:** зафиксировать минимальный UX просмотра графа узлов.

---

## 1) Scope

Клиент Node Browser v1:

* принимает на вход `node_id` (CID);
* выполняет fetch (SPEC-210);
* показывает поля Node (type/owner/created_at/policy) и ссылки `links`;
* позволяет переходить по ссылкам.

Клиент **ДОЛЖЕН** поддерживать просмотр истории версий по ссылкам:

* `rel="prev"`
* `rel="supersedes"`

---

## 2) Приватные Node

Если `policy.visibility=encrypted` и нет подходящего получателя/ключа:

* клиент **ДОЛЖЕН** показать “encrypted” без попыток расшифровки.

---

## 3) Диагностика

Клиент **ДОЛЖЕН** показывать:

* результат проверки CID/подписи (ok/invalid)
* источник/provider, откуда получен Node (если известен)

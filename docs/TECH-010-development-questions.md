# TECH-010: Вопросы для старта разработки (intake)

**Статус:** Draft (2026-02-02)  
**Назначение:** список вопросов, ответы на которые нужны перед началом реализации `ardents` по `spec/`.

---

## 1) Границы v1

1. Точно ли v1 = MVP-1 из `spec/SPEC-000-system-overview.md` (NET+MSG+AddressBook+Console), а остальное можно отложить?
2. Нужны ли в v1 `Node Graph`/`Providers`/`Encryption` (`SPEC-200/210/220`) или это MVP-2?

**Ответы (2026-02-02):**

1. Да. v1 = MVP-1 из `spec/SPEC-000-system-overview.md`: NET + MSG + AddressBook + Console. Всё остальное реализуется позже без переписывания ядра.
2. `Node Graph / Providers / Encryption` — MVP-2. В v1 допускаются только интерфейсы/типы без реализации.

---

## 2) Репозиторий и лицензирование

3. Проект open-source или private? Нужны ли `LICENSE`, `CONTRIBUTING`, `SECURITY.md`?
4. Нужен ли `CHANGELOG` и схема релизов (semver)?

**Ответы (2026-02-02):**

3. Рекомендуемый выбор: open-source с самого начала. Минимум: `LICENSE` (Apache-2.0 или MIT), `SECURITY.md` (короткий). `CONTRIBUTING` можно отложить.
4. `CHANGELOG` не нужен для v1: достаточно git history + теги. Semver вводится при появлении пользователей/внешних контрактов.

## 6) TLS/mTLS и идентификация peer

11. Где и в каком формате храним транспортный ключ и TLS cert (PEM vs keystore)?
12. Поддерживаем ли в v1 ротацию транспортного ключа (смена `peer_id`), или “ключ стабилен навсегда”?

**Ответы (2026-02-02):**

11. Хранение: транспортный ключ + TLS cert в PEM-файлах в `data/keys/` (без keystore).
12. В v1 транспортный ключ стабилен (ротация откладывается).

## 7) Библиотеки крипто/кодеков

13. Разрешено ли использовать внешние библиотеки для:
    - canonical CBOR,
    - dag-cbor,
    - CID/multihash/multibase,
    - xchacha20poly1305,
    - ed25519↔x25519 conversion/sealed-box?
14. Есть ли запреты/предпочтения по библиотекам?

**Ответы (2026-02-02):**

13. Да, внешние библиотеки разрешены для всего перечисленного.
14. Ограничения: библиотеки должны быть “живые” и иметь проверяемые режимы deterministic/canonical (без “магии” без тестов).

## 8) Хранилища и пути

15. Подтвердить структуру директорий: `config/`, `data/`, `run/` (в рабочем каталоге)?
16. Дедуп-таблица/кэш Node/ban-лист: хранить в памяти или на диске (переживать рестарт)?

**Ответы (2026-02-02):**

15. Да, структура фиксирована: `config/`, `data/`, `run/`.
16. В v1:
   - dedup table: in-memory
   - node cache: in-memory
   - ban list: in-memory
   - address book: disk
   Переживание рестарта не требуется.

**Ответы (2026-02-02):**

## 11) PoW параметры и бан-политика

20. Какой `default_difficulty` для dev и для prod (примерные числа)?
